## Architecture: Static Site Generation (SSG)

This site uses **Astro's default SSG mode** — all pages are pre-rendered to static HTML at build time. There is no server, no database, no API, no client-side data fetching.

```
Content (TypeScript data files)
         │
         ▼
  Astro components (.astro)
         │
         ▼
  Build: `astro build`
         │
         ▼
  ./dist/   ← static HTML/CSS/JS
         │
         ▼
  Netlify CDN → Browser
```

**Key implication:** To update content (e.g., add a new job or project), you edit a data file in `src/data/`, commit, and Netlify auto-deploys.

## Data Flow

All content is defined as typed TypeScript arrays in `src/data/`. Components import these and render them. The main page (`index.astro`) orchestrates the composition.

```
src/data/EXPERIENCE.ts
src/data/PROJECTS.ts       →  imported by components  →  rendered in index.astro
src/data/SIDE_PROJECTS.ts
src/data/RECOMMENDATIONS.ts
```

## How to Add Content

### Add a new job:

Edit [src/data/EXPERIENCE.ts](../src/data/EXPERIENCE.ts) — append a new object to the array following the existing shape.

### Add a new project:

Edit [src/data/PROJECTS.ts](../src/data/PROJECTS.ts) or [src/data/SIDE_PROJECTS.ts](../src/data/SIDE_PROJECTS.ts).

- Work projects go in `PROJECTS.ts` with `type: "work"`
- Personal projects go in `SIDE_PROJECTS.ts`
- Add a screenshot to `public/projects/` and reference it in the `image` field

### Add a blog post:

Create a new `.md` file in `src/pages/posts/`. Astro renders it automatically.

### Add a recommendation:

Edit [src/data/RECOMMENDATIONS.ts](../src/data/RECOMMENDATIONS.ts) — append a new object.

## No Caching, No Database, No APIs

This project has:

- **No server-side logic** — pure SSG
- **No database** — content is in TypeScript files
- **No external API calls** — no runtime data fetching
- **No caching layer** — Netlify CDN serves static files

## Mental Model

Think of it as: **"A typed CMS where the data files are the database, the components are the views, and `astro build` is the server."**

```
"Database" = src/data/*.ts (TypeScript arrays)
"Views"    = src/components/*.astro

```

## Practical Tips

1. **Content changes don't require touching components** — edit only the data files in `src/data/`.
2. **Type errors are caught at build time** — `pnpm build` runs `astro check` first.
3. **No JavaScript bundle by default** — Astro ships zero JS unless you add client directives (`client:load`, etc.). None are currently used.
4. **Images go in `public/`** — reference them with root-relative paths (e.g., `/projects/screenshot.png`).
5. **Prettier formats `.astro` files** — run `pnpm exec prettier --write .` to format.
6. **Hidden sections exist in index.astro** — "What am I looking for" section exists but is commented out.
7. **Tags on ExperienceItem are used for visual filtering context** — not functional filters currently.
